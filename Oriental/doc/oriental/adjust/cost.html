cost¶

provides cost functions to be used by oriental.adjust.

An important, concrete cost function is PhotoTorlegard, which allows for the definition of point observations in images from perspective cameras, where deviations from the perspective are modelled with Torlegard’s distortion parameters. For exemplary usage, see photo.

Besided PhotoTorlegard, cost provides various base classes from which users may derive concrete cost functors:

  • with AnalyticDiff, users need to compute residuals and their derivatives on their own;

  • with AutoDiff, users need to define only the method that computes residuals. Their derivatives are computed by automatic differentiation using that same method, supplying special objects of type Jet as arguments. For exemplary usage, see customCost.

PhotoTorlegard relates object and image points according to the specialization of the Spatial Similarity Transformation to the Perspective Transformation.

Instantiate PhotoTorlegard with your image observations as the first 2 arguments. Optionally, pass covInvSqrRoot, a 2x2 array for weighting, which is interpreted as the inverse square root of the cofactor sub-matrix of the observations: \(\\boldsymbol{Q_{ll}^{-\\frac{1}{2}}}\)

For setting the parametrization of the rotation matrix, set r.parametrization to one of the constants of EulerAngles

For lens distortion parameters (deviations from the pinhole camera model), see PhotoDistortion
