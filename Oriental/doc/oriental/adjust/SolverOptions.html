Solver.Options¶

Class Solver.Options provides means to configure the solver. While Solver.Options has numerous attributes, most of them affect the solver only if other attributes have been set appropriately. The graph below tries to explain these dependencies.

The root of the graph is Solver.Options, to be found on the lower left in red. Every other node (in yellow) represents one of its attributes. Edges indicate the dependencies among the attributes: a node at the head of an edge arrow affects the solver only if the resp. node at the edge’s tail does so, too. For better readability, attributes with the same dependencies have been merged into multi-row nodes, with one attribute per row.

The left column of every node (or of every row of multi-row nodes, resp.) denotes the attribute name. The right column specifies the default value of the attribute. If multiple values are given, then ‘→’ indicates the default value. Entries in the right column that are enclosed in angle brackets denote special values. Specifically, <EMPTY> and <NON-EMPTY> denote empty and non-empty lists or character strings, and <NUM-PROCS> stands for the number of available processors.

Important groups of nodes are enclosed in red boxes. Edge colours do not have significance.

Examplary dependencies:

  1. the choice of trust_region_strategy_type affects the solver only if minimizer_type is set to TRUST_REGION;

  2. the choice of dogleg_type in turn only affects the solver if trust_region_strategy_type affects the solver (see 1.), and if it has been set to DOGLEG
