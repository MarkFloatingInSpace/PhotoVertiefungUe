examples¶

Contents:

  • Photo observations
  • Custom cost functions

This short tutorial demonstrates basic usage of oriental.adjust and its sub-packages. As oriental.adjust’s core is based on Ceres, please read the Ceres tutorial before going through this one.

The central structure to define an adjustment problem is oriental.adjust’s class Problem, which binds Ceres’ class :Problem to Python. Basically, an instance of Problem manages a number of residual blocks, where each residual block produces one or more residuals, and depends on a cost function, a loss function, and 1 or more parameter blocks of arbitrary size. As the parameter blocks must not be copied but referenced by oriental.adjust, parameter blocks must be passed as numpy.ndarrays, and those numpy.ndarrays must outlive the Problem.

Various loss functions are built into oriental.adjust. While class (Trivial) allows for non-robust adjustments, other classes (Huber, Arctan, etc.) support robust adjustments.

Cost functions that seem useful in the field of object reconstruction are planned to be built into oriental.adjust.cost, as needed. Currently, the only built-in concrete cost function is PhotoTorlegard, with its usage being demonstrated in the example photo. Arbitrary cost functions may however be defined in Python, which is explained in the example customCost.

The code of all examples shown in this tutorial can be found in the folder $ORIENTAL/oriental/adjust/examples/.
