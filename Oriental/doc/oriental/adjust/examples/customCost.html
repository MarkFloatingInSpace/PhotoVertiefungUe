Custom cost functionsÂ¶

This example solves in Python the very basic adjustment problem which is solved in C++ in section Hello World! in the Ceres tutorial. The problem to solve is finding the minimum of the function

\[\frac{1}{2}(10 -x)^2.\]

Just like Ceres, oriental.adjust minimizes \(\frac{1}{2}\sum_{i=1} \rho_i\left(\left\|f_i\left(x_{i_1}, ... ,x_{i_k}\right)\right\|^2\right)\) by variation of the parameters \(x_{i_1}, ... ,x_{i_k}\). For non-robust adjustments using the Trivial loss (\(\rho_i(x) = x\)), this simplifies to \(\frac{1}{2}\sum_{i=1} \left\|f_i\left(x_{i_1}, ... ,x_{i_k}\right)\right\|^2\). Thus, for our example, we need to employ a cost function that computes the residual of \(10 - x\). As this cost function is not built into oriental.adjust, we need to define it in Python. One way to do this would be to derive from oriental.adjust.cost.AnalyticDiff. However, using AnalyticDiff, we would need to compute the cost functionâs derivatives w.r.t. the parameters on our own by analytic differentiation. Deriving from oriental.adjust.cost.AutoDiff instead, the derivatives are computed by oriental.adjust using automatic differentiation, and the definition of the cost function becomes very simple:

class CustomCost( adjust.cost.AutoDiff ):
    "Cost function that evaluates `10-x`"
    
    def __init__( self ):
        "Initialize an instance of CustomCost"
        
        numResiduals = 1
        parameterSizes = (1,)
        # initialize the base class
        super().__init__( numResiduals, parameterSizes )
        
    def Evaluate( self, parameters, residuals ):
        "Evaluate the ``residuals`` and possibly, their derivatives w.r.t. ``parameters``"

        # Store the data type of the ``parameters``.
        # It is identical to the data type of ``residuals``
        dt = type(parameters[0][0])
        
        # Compute the residual and possibly, also its derivatives
        # Use ``dt(10.)`` to ensure that the data type (possibly ``Jet``) assigned to ``residuals`` remains unchanged
        residuals[0] = dt(10.) - parameters[0][0]
        
        # indicate success
        return True
Following aspects in this class definition are important, and they must generally be considered when deriving classes from AutoDiff:

  1. CustomCost.__init__

    • AutoDiff must be initialized with the correct number of residuals that the cost function evaluates to. This information is passed to AutoDiff as the first argument to super( CustomCost, self ).__init__, as a scalar.

    • AutoDiff must be initialized with the correct number and sizes of the parameter blocks that the cost function depends on. This information is passed to AutoDiff as the second argument to super( CustomCost, self ).__init__, as a sequence (e.g. a tuple) of scalars. Thus, if the cost function depended on e.g. 2 parameter blocks of sizes 3 and 4, then the second argument would need to be (3,4)

  2. CustomCost.Evaluate

    • This method is called when solving the Problem. The arguments parameters and residuals will be sized as specified in the initialization of AutoDiff:

      • residuals will be a numpy.ndarray with numResiduals elements.

      • parameters will be a numpy.ndarray with as many elements as parameterSizes. Each item in parameters will again be a numpy.ndarray with as many elements as specified by the corresponding item in parameterSizes.

      In order to support automatic differentiation, each scalar in parameters and residuals will have either of 2 numpy.dtypeâs:

      • float_, when only the residual values shall be evalutated, or

      • Jet when additionally, their derivatives are needed.

      From the userâs point of view, this is of little importance, since many operators that exist for float_ are overloaded for Jet. However, one must make sure to retain the value type of residuals when assigning values to it. This can be done by querying the data type of either parameters or residuals and storing it in a local variable, as done in dt = type(param[0][0]). Whenever real numbers are used in the computation of residuals, they must first be converted to that data type, as in dt(10.)

Using CustomCost, the problem is solved easily using the following code:

def example_customCost():
    
    # The parameter to find the optimal value for.
    x = np.array([ 5.0 ])
    
    # Copy the original value for comparison with the found optimal value later on.
    x_init = x.copy()

    # Build the problem.
    problem = adjust.Problem()

    # Set up the only cost function (also known as residual).
    cost = CustomCost()
    
    loss = adjust.loss.Trivial()
    
    problem.AddResidualBlock( cost, loss, x )

    options = adjust.Solver.Options()
    
    summary = adjust.Solver.Summary()
    
    # Run the solver!
    adjust.Solve( options, problem, summary )

    print( summary.BriefReport() )
    
    print( "initial value: {}\n"
           "optimal value: {}".format( x_init[0], x[0] ) )
           
    assert summary.final_cost < summary.initial_cost, "adjustment failed"

The output should be:

Ceres Solver Report: Iterations: 2, Initial cost: 1.250000e+001, Final cost: 1.388518e-016, Termination: PARAMETER_TOLERANCE.
original value: 5.0
optimal value : 9.99999998334

The complete code used in this example can be found in $ORIENTAL/oriental/adjust/examples/customCost.py.
