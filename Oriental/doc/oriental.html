Aim¶

oriental is a Python package that aims at providing the functionality needed for manual and semi-automatic photogrammetric image orientation, camera calibration, and object reconstruction.

For tutorials on the usage of oriental.adjust, see examples.

Structure¶

The oriental package and its sub-packages and modules are implemented either

  • completely in Python, or they are

  • partly implemented in Python, and partly as C++ - extensions. C++ - extension package names start with an underscore (_) (e.g. oriental.log._log) and are sub-packages of their corresponding Python - packages (e.g. oriental.log). For convenience, all members of an eventual corresponding C++ - package are imported into an oriental Python - package upon import. Thus, all members of the C++ - sub-packages become available in their parent Python - module. However, the location from which such members originate will still be used in error messages, etc.

oriental makes use of established third-party Python packages, as far as this is reasonable. The most complete sub-package is currently adjust, which provides robust, non-linear least-squares adjustments. adjust._adjust is the Python-binding for the C++ - project oriental::adjust, which is based on Ceres. adjust._adjust binds most of Ceres’ functionality to Python, and defines further structures like cost functions (observation equations), loss functions, and distortion parameters. The definitions used in these additional structures largely follow those of ORIENT (“photo”, “ADP”).

redirectedPlotting¶

If True, then redirect plot commands to an oriental-IPython - subprocess, and immediately display plots in dedicated windows. If users choose to load oriental from an IPython shell without GUI loop integration, then we don’t redirect (either). Note: even if the IPython shell was startet with gui and matplotlib integration, it can be turned off temporarily using %run e.g. for efficient execution of an oriental-script Set to False for speedup. PyCharm is the only tested IDE that supports interactive plotting in the debugger, without oriental’s redirection.
