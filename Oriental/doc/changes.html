2016-06-30¶

  • new:
    • MonoScope:

      • allow for display/hiding of image points depending on their ObservationMode

      • point name suggestions:

        • namePrefix introduced.

        • minimum number of digits of suggested point names introduced.

        • order suggestions by smallest name / largest name / least recently digitized / most recently digitized.

        • warn if name is already used in image.

      • unified (asynchronous) reading of local files and WMS, with alpha channel, optionally with Lanczos interpolation.

      • track number of pending read requests and show presence in status bar.

      • replaced 3 different mouse modes with single mode, combined with keyboard modifier keys
        • Left mouse button: digitize a new or move an existing point, or pan the view.

          • Press <Alt> to pan the view.

          • Press <Ctrl> to force digitization, even when over another point label.

          • Press <Shift> to bring hidden points to the foreground.

        • Middle mouse button: pan the view.

        • Mouse wheel: zoom the view.

        • Right mouse button on point: point context menu.

      • general image overlays introduced, with selectable opacity and optionally maximized contrast.

      • disallow movement of image points into invalid image areas.

      • upon loading images, ask user if overviews shall be generated, if missing.

      • restrict the sensitive area of image points and their names to the actually drawn area instead of their bounding window.

      • show both mouse position and value of overlay’s first channel (if available) in the status bar.

      • set maximum tile cache size upon startup to half the available physical memory.

      • support $ORIENTAL in DB image paths.

      • when opening image files interactively, don’t restrict their file extensions.

      • syntax highlight the XML in WMS dialog.

      • whatsThis introduced.

      • SQL command window.

  • improved:
    • MonoScope:

      • same point name suggestions whether point name confirmation is active or not.

  • updated:

    • Python 3.5.2

    • three.js r78

    • ExifTool 10.20

    • PyCharm 2016.1.4

2016-05-19¶

  • new:

    • GetParameterBlocks(), GetParameterBlocksForResidualBlock(): return original numpy ndarrays, so potentially attached information is retained.

    • PhotoTorlegard: allow for attaching arbitrary Python objects as attribute data, which will be retained.

    • MonoScope:

      • indirectly measure an image point as the intersection of 2 straight lines.

      • for mosaics created with absOri, provide the option to alternatively display the number of overlapping aerials. Use hotkey ‘O’ to toggle the display.

      • asynchronous WMS requests - keeping MonoScope responsive while waiting for server response.

      • customizable size of image point displays (text and disc).

      • support copying tables to clipboard.

      • support GDAL WMTS-driver

      • full integration with the OrientAL logging system

    • EvaluateOptions: new attribute weighted tells cost functions to consider their weights or not during Evaluate().

    • relOri: if Exif provides no PRC positions, then try to use LBA for datum transformation.

    • absOri:

      • fully support scanned images.

      • export image footprints: for each aerial image, produce a shape file with the image corners back-projected onto the assumed horizontal ground plane.

      • in addition to mosaic.tif created via homographies, export mosaicOverlap.tif, which stores the number of overlapping images for each pixel of mosaic.tif.

    • ortho:

      • fully support scanned images.

      • in addition to raw DSM and orthophoto, provide interpolated DSM and refined orthophoto.

      • in addition to orthophoto mosaics, produce a classic (non-true) and / or true orthophoto for each aerial image. Can be deactivated with new command-line options –no-separateTrueOrthos and –no-separateClassicOrthos.

    • customizable web mapping services as XML files in the data directory.

    • ship with PyCharm

  • improved:

    • Set the GDAL WMS-cache directory via an environment variable instead of in dataset XML-files

    • mergeTiles(): support merging files with different bit depth. This is necessary, because for void areas, SURE produces 16bit files with invalid pixels only, even if all input images are 8bit.

  • updated:

    • DB-layout change: imgobs.name’s type affinity is now TEXT and its value may be NULL, because with the former type affinity INTEGER, SQLite cannot be forced to store ‘08’ as TEXT, but it will cast it to 8. imgobs.name will from now on be considered as image point attribute that is relevant for display in MonoScope only and possibly for custom scripts. imgobs.name may not be unique within the point’s image, which may be the case e.g. in the overlapping areas when manually measuring GCP image observations in overlapping sub-blocks (-> multiple, independent image observations of the same object point in the same image). No more trigger that automatically casts imgobs.id to imgobs.name if imgobs.name is NULL. For unique identification of an image observation, use imgobs.id instead of the tuple (imgobs.imgid,imgobs.name) from now on.
