Installation¶

Supported Operating Systems¶

OrientAL is tested to run on x64-installations of Microsoft Windows 10 and on Ubuntu 18.04 LTS. OrientAL may run on other versions of Windows and Ubuntu or other Linux distributions, but this is not tested.

Prerequisites for Installations on Windows¶

The latest Microsoft Visual C++ Redistributable Package for Visual Studio 2019 and the x64-architecture needs to be installed on the PC where OrientAL shall be executed. Repeated installations of this package will do no harm to your PC. Thus, if unsure whether this package has already been installed, just execute the respective installer in order to avoid trouble. The installer can be downloaded from Microsoft. For your convenience, it comes bundled with OrientAL, to be found here.

Should this package be missing from your PC, then you will encounter the infamous (since unspecific) error DLL load failed: The specified module could not be found.

Procedure¶

Windows¶

OrientAL is shipped as the single self-extracting archive OrientAL_<build-date><build-options>.exe. Extract the archive by double-clicking onto it. In the dialog that opens, select an empty directory of your choice into which your installation of OrientAL shall be extracted. This manual refers to that directory as $ORIENTAL. Having confirmed the dialog, your installation of OrientAL becomes available there. Once you have put your license file into the proper sub-directory, your installation of OrientAL is ready. OrientAL is portable software, which can therefore be moved and executed to/in any other location of any other PC, without re-extracting the archive.

Ubuntu¶

OrientAL is shipped as the compressed archive oriental_<build-date><build-options>.tar.xz, together with the similarly named installation script oriental_<build-date><build-options>_install.sh. Place the archive and your License file into the same directory of your choice and execute the installation script in a shell. The installation directory defaults to ./oriental, but may be given as command-line argument. Additionally, the installation script installs necessary 3rd-party packages and copies your license file to the proper location.

License file¶

To be able to run OrientAL, the file oriental.key needs to be present in the directory $ORIENTAL/data. Contact wilfried.karel@geo.tuwien.ac.at to receive it.

OrientAL shell¶

To setup the appropriate environment and avoid errors due to other installed software, it is strongly recommended to start the OrientAL Python interpreter and all OrientAL scripts and executables only from within an OrientAL shell. To start this shell, double-click onto $ORIENTAL/OrientALShell.bat (Windows) or $ORIENTAL/OrientAL (Ubuntu).

As an exception, MonoScope may be started directly by double-clicking onto $ORIENTAL/MonoScope.bat (Windows) or $ORIENTAL/MonoScope (Ubuntu).

User scripts¶

The OrientAL shell executes its arguments, if present. Thus, users may e.g. prepare process.bat as custom batch file on Windows and execute it from a standard Windows console as:

OrientALShell.bat process.bat

This executes process.bat within the OrientAL shell.

Installation of Additional Python Packages¶

pip supports the administration (installation, upgrade, etc.) of Python package distributions, and it is among the Pre-Installed 3rd-Party Python Packages. Such Python packages may consist of Python and/or C/C++ source code and/or binaries. Binaries, however, must be shipped in the wheel format. If the Python package to be installed consists of Python source code only, or if its repository provides a binary distribution in the wheel format, then its installation can easily be done within an OrientAL shell:

python -m pip install PySide

would e.g. download a (binary) wheel of the latest stable version of PySide, a Python-binding for Qt, via the Python Package Index (PyPI), and add it to the Python installation used by OrientAL.

Keeping OrientAL Portable on Windows¶

Additional Python packages are installed into the directory $ORIENTAL/python/Lib/site-packages/. However, depending on the package, the installation procedure may also install package scripts into the directory $ORIENTAL/python/Scripts/ in addition to the package itself, which pip reports in its output:

Installing ... script to ...\python\Scripts

Note that package scripts installed this way will not be portable: having moved your installation of OrientAL to another location, calling such an additional package script will either invoke the wrong Python interpreter (python.exe), thus affecting another Python installation on your PC, or result in the not-so-helpful error message:

failed to create process.

In order to overcome this problem, open the respective package scripts (matching $ORIENTAL/python/Scripts/*-script.py) in a text editor. You will find the absolute file path to python.exe at the time when pip.exe installed the package, e.g.:

#!C:\OrientAL\python\python.exe

Simply remove the directories, leaving only python.exe:

#!python.exe

save the script file, and try to run the script again.
